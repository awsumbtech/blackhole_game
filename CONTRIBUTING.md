# Contributing to Black Hole: Galaxy Eater

Thank you for your interest in contributing! This project welcomes contributions from everyone.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/blackhole_game.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test thoroughly (open `index.html` in a browser)
6. Commit with clear messages: `git commit -m "Add: description of your changes"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Development Setup

No build tools required! Simply:
- Open `index.html` in a modern browser (Chrome, Firefox, Safari, Edge)
- Or run a local server: `python -m http.server 8000` or `npx serve`

## Code Style

- Use ES6+ features (modules, arrow functions, const/let)
- Keep functions small and focused
- Add comments for complex logic
- Use descriptive variable names
- Follow existing code structure

## Areas for Contribution

### Gameplay
- New object types with unique behaviors
- Additional biomes and themes
- Power-up system
- Boss entities or special encounters
- Achievement system

### Audio
- New procedural sound effects
- Music system
- Audio visualization
- Spatial audio effects

### Visual
- Particle effect improvements
- New visual themes
- Animation polish
- Visual feedback enhancements
- Shader effects (if adding WebGL)

### Technical
- Performance optimizations
- Mobile control improvements
- Accessibility features (keyboard navigation, screen reader support)
- Internationalization (i18n)
- Save system enhancements

### Documentation
- Tutorial system
- Better code documentation
- Video guides
- Translation of README

## Testing

Before submitting:
1. Test in multiple browsers (Chrome, Firefox, Safari)
2. Test on mobile devices (iOS Safari, Chrome Mobile)
3. Verify no console errors
4. Test with audio on/off
5. Test save/load functionality
6. Ensure game remains playable through multiple galaxies

## Pull Request Guidelines

- Keep PRs focused on a single feature or fix
- Include a clear description of changes
- Reference any related issues
- Update README.md if adding features
- Test thoroughly before submitting
- Be responsive to feedback

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Focus on the code, not the person
- Help newcomers learn
- Be patient with reviews

## Questions?

- Open an issue for discussion
- Check existing issues first
- Be specific and provide examples
